# Skills Hub Web 设计规范

> 适用范围：`apps/web/**` 与 `packages/shared/src/components/**` 中服务 Skills Hub Web 的共享组件。
> 本规范是 Web 端专门规范；桌面应用端继续遵循 `docs/DESIGN.md` 与 `docs/design-spec.md` 的应用端章节。

## 1. 设计方向

Skills Hub Web 参考 `https://skillhub.cn` 的公开页面风格，但保留 mclaw 自身品牌、信息架构和业务文案。

核心气质：

- 白色画布，而不是暖奶油全站底色。
- 搜索优先，市场页以搜索条和技能榜单流为第一任务。
- 黑色胶囊按钮用于主操作。
- 蓝色用于标题重点、评分/状态重点和提示面板。
- 卡片弱化，列表强化，避免过度 bento 化。
- 内容保持 mclaw 自有表达，不复制参考站长文案或品牌内容。

## 2. 颜色

| 用途 | Token / 类名 | 色值 | 说明 |
|------|--------------|------|------|
| 画布 | `--background` | `#FFFFFF` | Web 主背景 |
| 主文本 | `--foreground`, `skillhub.ink` | `#1C1C1E` | 标题、正文主色 |
| Web 强调蓝 | `--primary`, `skillhub.blue` | `#3957FF` | Hero 强调、状态重点、评分星 |
| 黑色 CTA | `skillhub.black` | `#202020` | 主按钮、选中导航、安装按钮 |
| 细线 | `--border`, `skillhub.line` | `#E6E9EF` | 搜索框、卡片、分割线 |
| 浅底 | `--secondary`, `skillhub.soft` | `#F6F7FB` | 标签、功能区、轻面板 |
| mclaw 橙 | `brand` | `#EE7C4B` | 品牌辅助色，不能作为 Web 主强调 |

### 2.1 品牌橙色落地清单

橙色在 Skills Hub Web 中只做**品牌锚点**，不做功能强调。以下是具体使用位置和用法：

#### 必须出现橙色（品牌标识层）

| 位置 | 用法 | 规格 |
|------|------|------|
| **Logo / 品牌标识** | 橙色方块或图标 + "mclaw" 文字 | 标识主色 `#EE7C4B`，不放蓝色 |
| **favicon / Open Graph 图** | 橙色底 + 白色图形 | 浏览器标签页身份锚点 |
| **页脚品牌区** | 小号 Logo + "Powered by mclaw" | 橙色图标点缀，文字灰色 |

#### 建议出现橙色（场景过渡层）

| 位置 | 用法 | 规格 |
|------|------|------|
| **「在桌面端打开」入口** | 橙色边框按钮或橙色图标 | 暗示「回到工具」，和蓝色主 CTA 区分 |
| **安装成功反馈** | 橙色勾选动画 / 橙色轻背景 | 正向情绪反馈，区别于蓝色信息提示 |
| **桌面端功能引用** | 文案旁橙色小圆点或标签 | 如「Pro 版桌面端可用」标签 |

#### 禁止使用橙色（避免混淆层）

| 不要 | 原因 |
|------|------|
| 主 CTA 按钮用橙色 | 和蓝色 CTA 抢注意力，用户困惑哪个是主操作 |
| 导航激活态用橙色 | 导航选中 = 黑底白字胶囊，橙色破坏一致性 |
| 评分 / 状态标签用橙色 | 这些用蓝色 `#3957FF`，橙色留给品牌元素 |
| 链接 / 可点击文本用橙色 | 链接用蓝色或黑色，橙色文字在白底上对比度差 |
| 大面积橙色背景或渐变 | Web 是白底画布，橙色大面积会像警告页 |

#### 橙色和蓝色的协作规则

```
用户路径示例：
  进入首页 → 看到橙色 Logo（品牌认知）
  → 蓝色搜索条聚焦（开始任务）
  → 黑色胶囊安装（主操作）
  → 橙色安装成功提示（品牌温度）
  → 页脚小橙色 Logo（收尾锚点）
```

核心原则：**蓝色引路，黑色拍板，橙色只在入口和出口出现。** 橙色是「mclaw 的签名」，不是「操作的指引」。

## 3. 字体

| 用途 | 字体 |
|------|------|
| 正文与 UI | `Plus Jakarta Sans`, `-apple-system`, `SF Pro Text`, `PingFang SC`, `sans-serif` |
| 展示标题 | `Outfit`, `SF Pro Display`, `PingFang SC`, `sans-serif` |
| 命令与代码 | `JetBrains Mono`, `ui-monospace`, `SF Mono`, `monospace` |

标题不使用负字距。中文正文保持 14-16px，行高 1.5-1.7。

## 4. 布局

- 顶栏：`68px` 高，白色半透明背景，底部 `border-black/[0.06]`，固定在顶部。
- 首页首屏：居中 `max-w-[960px]`，桌面顶部留白约 `140px`，展示标题桌面约 `58px/1.05`。
- 宽容器：营销/首页模块可到 `max-w-[1440px]`。
- 内容容器：列表、详情、定价页使用 `max-w-[1180px]`。
- 技能市场：标题区 + 圆形搜索条 + 分类胶囊 + CLI 提示 + 纵向技能流。
- 移动端：按钮可堆叠，导航隐藏次要入口，禁止横向溢出。

## 5. 组件规范

### 5.1 顶栏

- Logo 使用 mclaw 名称和蓝色方形标识。
- 导航项为胶囊：选中黑底白字，未选透明/浅底 hover。
- 顶栏不放“发布 Skill”；发布/审核属于管理后台能力。
- 登录按钮为黑色胶囊。
- 管理后台入口只在已登录管理员账号的用户菜单中显示。

### 5.2 按钮

主按钮使用 `.skillhub-capsule-button`：

```css
rounded-full;
background: #202020;
color: #fff;
padding: 12px 20px;
```

次按钮使用 `.skillhub-ghost-button`：白底、黑色 10% 边框、胶囊圆角。

### 5.3 搜索与筛选

- 搜索框桌面高度 `54px`，移动端 `48px`。
- 使用 `rounded-full`，白底，`#E6E9EF` 边框。
- 允许轻投影：`0 8px 24px -4px rgb(0 0 0 / 5%)`。
- 分类标签使用胶囊；选中态黑底白字。

### 5.4 技能列表

- 默认使用纵向横向条目，不使用三列卡片网格。
- 列表项：白底、`rounded-[24px]`、`border-black/[0.06]`。
- hover：轻微上移、边框变深、投影 `shadow-xl shadow-black/5`。
- 图标：`48px`、`rounded-2xl`、浅底蓝字；无图标时用技能名首字。
- 描述最多 3 行，标签和评分放在底部信息行。
- “发布 Skill”按钮只对管理员显示，普通用户和未登录用户不显示。

### 5.5 详情页

- 头部使用大白色面板：`rounded-[32px]`、浅边框。
- 安装按钮使用黑色胶囊。
- 标签、评价、版本历史使用浅灰底和轻分割线。
- 安装弹窗使用白底大圆角，命令块使用浅灰底。

### 5.6 定价页

- 定价卡：`rounded-[28px]`、白底浅边框。
- Pro/推荐卡：浅蓝底 `#F7F8FF` + 蓝色细线。
- CTA：推荐方案使用黑色胶囊，其他方案使用白底边框胶囊。

## 6. 反模式

| 不要 | 要 |
|------|----|
| 全站使用暖奶油画布 | 使用白色画布，局部浅灰/浅蓝 |
| Web 主操作继续用 mclaw 橙 | 主操作用黑色，强调用蓝色 |
| 默认三列技能卡片 | 默认纵向技能榜单流 |
| 小圆角矩形搜索框 | 圆形搜索条 |
| 大面积阴影和重卡片 | 浅边框 + 轻 hover 投影 |
| 复制 skillhub.cn 文案/品牌 | 借鉴视觉系统，保留 mclaw 内容 |
| 把发布技能放在公开顶部导航 | 发布入口仅在管理员已登录后的管理后台/技能市场中出现 |
