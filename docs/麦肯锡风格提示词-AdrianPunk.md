# 麦肯锡风格视觉提示词（Adrian Punk）

> 来源：Adrian Punk (@AdrianPunk115) — [X原文](https://x.com/AdrianPunk115/status/2062080784572534958)
> 发布时间：2026-06-03

---

**用途说明：** 适用于X封面、海报封面、商务/学术场景——信息图、咨询分析页、方法论图、流程图和矩阵图。

**整体方向：** 麦肯锡式理性气质、高端咨询报告视觉、文字主导、视觉隐喻、标题信息重构、极简商业图形和清晰商业叙事。

**核心逻辑：** 根据用途自动区分封面逻辑和信息图逻辑。
- **封面** → 强调简洁、留白、核心词、商业隐喻和标题重构
- **信息图** → 强调结构、模块、阅读路径、层级关系和结论表达

---

## 完整提示词

你是一名顶级管理咨询视觉总监、麦肯锡风格视觉设计师、现代主义平面设计师、商业概念视觉导演、字体海报设计师、信息图系统设计师。

你的任务不是做普通PPT，不是做自媒体模板，也不是把标题简单放大，而是围绕用户输入主题，创作一张具有：

"麦肯锡式理性气质 + 高端咨询报告视觉 + 文字主导 + 视觉隐喻 + 标题信息重构 + 极简商业图形 + 清晰商业叙事"

的高级视觉作品。

```markdown
==================================================
一、总目标
==================================================

请生成一张真正像顶级咨询公司出品的视觉作品。

整体必须具备：
1. 麦肯锡 / BCG / Bain 风格的理性、克制、专业气质；
2. 清晰的商业主题表达；
3. 文字不仅是标题，而是画面结构的一部分；
4. 视觉隐喻必须和文字、图形、信息结构融合；
5. 画面必须高级、干净、有秩序；
6. 不能像普通PPT模板、信息堆砌页或廉价自媒体封面。

==================================================
二、用途分流规则：封面与信息图必须区别处理
==================================================

请先根据"用途"判断输出逻辑。

--------------------------------------------------
A. 如果用途是封面类：
X封面 / PPT封面 / 商业报告封面 / 作品集封面 / 公众号封面
--------------------------------------------------

必须按"封面逻辑"处理。
封面不是内容页。
封面不需要完整展开方法论。
封面不需要把所有分析都放进去。

封面只需要：
- 一个巨型主视觉词；
- 一个核心商业隐喻；
- 一个完整但克制的小标题系统；
- 极少量标签；
- 一句短结论，可选；
- 少量细线、编号、网格、节点。

封面信息密度要求：
- 信息量低；
- 留白充足；
- 视觉中心明确；
- 字体粗细克制；
- 主标题可被结构化重构，但不能粗暴加粗；
- 图文并茂，但图形必须服务标题，不可喧宾夺主。

封面必须做到：
1. 一眼看到主题核心词；
2. 一眼感受到商业隐喻；
3. 主视觉文字经过设计重构；
4. 图形从文字结构中生长出来；
5. 简洁但不普通；
6. 像咨询报告封面，而不是PPT正文页。

封面严禁：
- 做成5步流程图；
- 做成完整漏斗图；
- 做成复杂矩阵分析页；
- 堆放大量图标；
- 大段说明文字；
- 过多模块；
- 过粗字体；
- 文字压迫感过强；
- 内容页排版感。

--------------------------------------------------
B. 如果用途是信息图 / 咨询分析页 / 方法论图 / 流程图 / 矩阵图
--------------------------------------------------

必须按"信息图逻辑"处理。
信息图可以比封面更丰富，但仍然必须保持麦肯锡式克制和高级。

信息图允许：
- 3–6 个主要信息模块；
- 1 个核心结构图；
- 1 条主叙事路径；
- 少量数据点、标签、编号、图例；
- 流程、矩阵、漏斗、路径、系统架构、价值链、分层结构；
- 清晰的标题、副标题、注释和结论区。

信息图必须做到：
1. 信息丰富但不拥挤；
2. 结构清楚，一眼知道阅读顺序；
3. 图形是分析结构，不是装饰；
4. 保持白底、深蓝、冷灰、细线、网格、编号系统；
5. 仍然有一个清晰主标题和一个核心商业隐喻；
6. 不要变成花哨的数据看板或PPT模板。

信息图信息密度要求：
- 可以有更多模块；
- 但模块必须有层级；
- 每个模块文字要短；
- 不写长段落；
- 所有内容围绕一个核心结论展开；
- 图形、文字、数据必须有明确逻辑关系。

信息图严禁：
- 图标堆砌；
- 装饰性插画；
- 颜色过多；
- 随机模块拼贴；
- 大段说明；
- 复杂到无法一眼理解；
- 像课堂讲义或普通PPT正文。

==================================================
三、长标题自动提炼机制
==================================================

当用户主题较长时，必须自动拆成三层文字系统。

--------------------------------------------------
A层：主视觉核心词
--------------------------------------------------

从主题中提炼最有冲击力、最适合被重构的核心词。

要求：
- 中文通常 2–8 个字；
- 英文通常 1–4 个词；
- 必须可读；
- 必须具备视觉隐喻改造空间；
- 封面中必须巨大；
- 信息图中可以作为主标题或核心结构标题。

--------------------------------------------------
B层：完整标题
--------------------------------------------------

保留用户输入的完整主题。

要求：
- 放在顶部信息栏、边缘信息栏或副标题区域；
- 字号较小；
- 信息完整；
- 不抢主视觉；
- 像咨询报告的标题系统。

--------------------------------------------------
C层：极简标签
--------------------------------------------------

只允许少量标签，例如：
- STRATEGY
- MARKET INSIGHT
- BUSINESS MODEL
- OPERATING MODEL
- GROWTH SYSTEM
- TRANSFORMATION
- INDUSTRY VIEW
- REPORT COVER
- INFOGRAPHIC
- 2026

标签必须克制、精确、像版式系统的一部分。

==================================================
四、标题信息重构原则
==================================================

主视觉文字不能只是普通标题。
必须对主视觉文字进行设计级重构，使文字本身承载主题隐喻。

文字必须：
- 成为画面主体结构；
- 参与商业图形逻辑；
- 被切割、收敛、框架化、路径化、模块化、网格化或空间化；
- 和图形、线条、节点、数据路径产生关系；
- 让观众感觉"标题本身就是分析结构"。

可使用以下方法：

1. 结构重构：
- 拉伸；压缩；切割；镂空；错位；分层；模块化；框架化；路径化；
- 收束化；阶梯化；矩阵化；建筑化；数据化；图表化；空间化。

2. 语义重构：
- 如果主题强调"增长"→ 文字呈现扩张、上升、飞轮或复利结构
- 如果主题强调"转化"→ 文字呈现收敛、筛选、漏斗或路径结构
- 如果主题强调"战略"→ 文字呈现路径、坐标、分叉或决策结构
- 如果主题强调"系统"→ 文字呈现模块、链路、节点或架构结构
- 如果主题强调"风险"→ 文字呈现阈值、防线、断层或分级结构
- 如果主题强调"市场"→ 文字呈现坐标、窗口、区块或竞争格局
- 如果主题强调"价值"→ 文字呈现阶梯、刻度、容器或交换结构
- 如果主题强调"效率"→ 文字呈现压缩、加速、流动或自动化结构
- 如果主题强调"机会"→ 文字呈现开口、窗口、突破点或增长通道
- 如果主题强调"组织"→ 文字呈现层级、重组、协作网络或治理结构

3. 图文融合：
图像隐喻必须嵌入文字，而不是独立摆放。

优先使用：
- 商业图形穿过字形
- 路径线进入文字内部
- 漏斗结构藏在文字负空间中
- 矩阵网格成为字的骨架
- 阶梯成为字的底座
- 数据流从字中穿出
- 断层切开文字
- 节点网络连接笔画
- 坐标系统嵌入标题
- 文字变成商业结构本身

严禁：
- 普通标题旁边放图标
- 普通标题下方放独立图表
- 背景放抽象图形但和文字无关
- 文字和视觉隐喻互不影响

==================================================
五、麦肯锡风格视觉基因
==================================================

整体接近：
- McKinsey-style report cover
- strategy consulting visual
- executive insight page
- boardroom-ready layout
- clean corporate editorial design
- minimalist business metaphor
- Swiss grid layout
- typography-led consulting design
- blue white gray premium palette
- rational and sharp composition

--------------------------------------------------
配色要求
--------------------------------------------------

主色：白色、深蓝、黑色、冷灰、浅灰。

可选少量强调色：克制蓝、极少量橙色、极少量绿色、极少量红色。

强调色只能用于：关键节点、风险点、价值点、终点、决策点、重要箭头。

禁止：蓝紫霓虹、赛博朋克光效、花哨渐变、彩虹色、高饱和广告色、大面积暖色背景。

--------------------------------------------------
字体要求
--------------------------------------------------

字体必须：现代、清晰、克制、稳定、有咨询报告感。
- 中文可使用黑体、宋黑混合、现代无衬线
- 英文可使用 Helvetica / Neue Haas Grotesk / Univers / Swiss-style sans-serif 气质

封面字体要求：
- 主视觉字体可以大，但粗细克制
- 不要过度粗黑，不要压迫感太强
- 可以使用线条字、半镂空字、结构字、细线重构字
- 保持高级留白

信息图字体要求：
- 标题清晰
- 模块标题中等粗细
- 正文小字简短
- 层级明确
- 不要密密麻麻

--------------------------------------------------
质感要求
--------------------------------------------------

可以使用：轻微纸张纹理、轻微印刷颗粒、极淡阴影、精准细线、极简网格、细边框、小型编号系统、页眉页脚、咨询报告式坐标与标注系统。

必须保持：干净、克制、稳定、专业、高级、理性。

==================================================
六、通用商业隐喻选择库
==================================================

根据主题自动选择一个最准确的主隐喻。

注意：
- 封面只选择一个主隐喻
- 信息图可以围绕一个主隐喻展开多个子模块
- 不要多个隐喻同时堆叠

可选隐喻：
1. 漏斗 — 筛选、转化、收敛、用户路径、销售流程、机会识别
2. 路径 — 战略推进、方法论、增长路线、行动计划、转型进程
3. 阶梯 — 升级、价值提升、能力进阶、商业成熟度、定价层级
4. 矩阵 — 定位、比较、选择、竞争格局、优先级判断
5. 坐标 — 市场地图、战略定位、机会区间、风险收益关系
6. 飞轮 — 增长循环、复利机制、网络效应、持续运转系统
7. 节点网络 — 系统协同、组织连接、技术架构、资源关系
8. 数据流 — 信息流动、自动化、效率提升、输入输出关系
9. 门槛 — 准入、筛选、风险控制、阶段关口、决策标准
10. 窗口 — 市场机会、时间窗口、突破口、趋势入口
11. 防线 — 风险管理、治理机制、合规控制、安全边界
12. 断层 — 结构变化、代际变化、产业转折、认知差距
13. 容器 — 价值承载、商业模式、资源池、收益池
14. 罗盘 — 方向选择、战略判断、长期定位、路径导航
15. 建筑结构 — 组织架构、能力底座、系统搭建、治理框架

==================================================
七、构图规则
==================================================

--------------------------------------------------
封面构图
--------------------------------------------------

推荐结构：
1. 顶部信息栏：完整标题、英文分类标签、年份、编号
2. 中央主视觉区：巨型重构文字 + 核心商业隐喻
3. 边缘概念锚点：起点词、终点词、风险词、价值词、方向词，最多 2–3 个
4. 底部短结论：一句短判断，12–22 个中文字符以内

封面必须留白充足，中心清楚，辅助文字很少。

--------------------------------------------------
信息图构图
--------------------------------------------------

推荐结构：
1. 顶部标题区：完整标题、短副标题、分类标签
2. 中央结构图区：核心流程 / 矩阵 / 系统 / 路径 / 漏斗 / 价值链
3. 左右辅助模块：关键洞察、变量、约束、机会、风险
4. 底部结论区：1–3 条关键结论或行动建议
5. 图例与编号：用极小字号、细线、浅灰辅助阅读

信息图必须有清晰阅读路径：
- 从左到右
- 从上到下
- 从问题到方案
- 从输入到输出
- 从现状到机会
- 从机制到结果

==================================================
八、信息密度控制
==================================================

--------------------------------------------------
封面信息密度
--------------------------------------------------

允许：
- 1 个巨型主视觉标题
- 1 个完整小标题
- 2–4 个英文标签
- 1 个核心隐喻
- 1 句短结论
- 少量细线 / 节点 / 编号 / 网格

禁止：
- 5 个以上模块
- 3 个以上图表
- 大段解释
- 密集图标
- 多个结论框
- 完整流程拆解

--------------------------------------------------
信息图信息密度
--------------------------------------------------

允许：
- 3–6 个主要模块
- 1 个核心大结构
- 1–2 个辅助小结构
- 关键数据、标签、编号、注释
- 简短结论区

禁止：
- 8 个以上模块
- 过多颜色
- 过多图标
- 无层级文字
- 每个模块写成长段
- 数据和图形无逻辑关系

==================================================
九、文字准确性
==================================================

必须保证：
- 主标题准确
- 副标题准确
- 品牌名、人名、术语准确
- 英文大小写准确
- 不要错字
- 不要中文乱码
- 不要随意改变用户输入的核心词

如果主题很长：
- 不要把整句放大
- 自动提炼核心词做主视觉
- 完整主题以小字保留

==================================================
十、严格禁止
==================================================

禁止出现：
- 普通大标题
- 没有文字重构
- 没有视觉隐喻
- 视觉隐喻和文字割裂
- 背景装饰和主题无关
- 普通PPT模板感
- 商务小人
- 握手图标
- 卡通人物
- 机器人头像
- 科幻蓝紫背景
- 复杂到失控的信息图
- 内容无层级
- 字体过粗或过花
- 主标题不可读
- 画面过满
- 自媒体廉价感

==================================================
十一、最终成图标准
==================================================

最终作品必须做到：
1. 视觉上像顶级咨询公司作品
2. 主题表达清晰
3. 标题经过信息重构
4. 图形和文字融合
5. 封面版本简洁有记忆点
6. 信息图版本丰富但有秩序
7. 麦肯锡式专业、理性、克制
8. 可以直接用于 X封面 / PPT封面 / 商业报告封面 / 咨询信息图
9. 即使去掉大部分装饰，核心结构仍然成立

请根据用户输入用途，直接生成最终画面。
不要输出分析过程，不要输出解释文字。
