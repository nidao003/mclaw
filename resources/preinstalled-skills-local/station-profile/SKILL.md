# 站点画像展示指导

## 数据板块

1. **基本信息**：站名、所属线路、城市、行政区
2. **客流**：日均、高峰时段、趋势
3. **人口**：800 米半径内常驻/工作/到访人口
4. **商业**：周边 POI、商圈类型、商业活跃度
5. **投放价值**：可用广告位、受众画像

## 呈现顺序

- 以最贴合用户问题的指标开头
- 提供对比上下文（城市均值对比、排名）
- 以可执行的洞察或建议收尾

## 跟进建议

展示站点数据后，给出 3 个跟进方向：
1. 深入某个指标
2. 与另一车站对比
3. 生成报告

## 资源 ID 保留（强制）

每次输出站点画像后，**必须保留**本次涉及车站的资源 ID：`stationId` / `cityCode` / `cityId` / `lineId`。

- 单站点画像：保留该站完整四元组
- 多车站对比：按正文中提及顺序，为每个车站保留完整四元组
- 所有 ID 必须直接来自 `query.py` 脚本返回（`search_stations`、`station_profile`、`line_info` 等），**禁止凭记忆或猜测填写**

> 这些 ID 是前端渲染站点卡片、跨轮上下文的关键。即使用户可见的文字省略了，结构化结果里也不要丢弃。
